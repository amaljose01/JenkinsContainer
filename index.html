Install Jenkins

Running Jenkins Container

Bash
docker run -d --name jenkins \ -p 8080:8080 -p 50000:50000 \ -v jenkins_home:/var/jenkins_home \ jenkins/jenkins:2.528.3-jdk21
Jenkins will be available at http://localhost:8080

What This Does

Runs container in the background

Maps container port 8080 to localhost:8080 (Jenkins UI)

Persists Jenkins data (configs, plugins, build history)

Create Git-Managed CI Images

Reusable Docker images for your CI/CD pipelines

Why Version Control Images? Track changes to build environments, ensure reproducibility, and share configurations across teams

Python Dockerfile

Dockerfile
FROM python:3.11-slim RUN pip install --no-cache-dir \ requests \ pandas \ numpy WORKDIR /workspace

Node Dockerfile

Dockerfile
FROM node:20-slim WORKDIR /workspace

Build Script

ci-images/build-images.sh
#!/bin/bash set -e echo "Building Python image..." docker build -t ci/python-runner:1.0 ./python echo "Building Node image..." docker build -t ci/node-runner:1.0 ./node echo "All images built successfully"
Directory Structure
ci-images/ ├── python/ │ └── Dockerfile ├── node/ │ └── Dockerfile └── build-images.sh
Build Command
cd ci-images chmod +x build-images.sh ./build-images.sh
Verify Images
docker images | grep ci/

Should show:

ci/python-runner:1.0
ci/node-runner:1.0

Create Jenkins Pipelines

Build, test, and deploy with Kubernetes pods

  • Multi-Container
  • Python Only

Multi-Container Pipeline (Python + Node)

Groovy
pipeline { agent { kubernetes { yaml """ apiVersion: v1 kind: Pod spec: containers: - name: python image: ci/python-runner:1.0 command: ['cat'] tty: true - name: node image: ci/node-runner:1.0 command: ['cat'] tty: true """ } } stages { stage('Build') { steps { container('node') { sh 'npm install' } container('python') { sh 'python scripts/run.py' } } } } }

Python-Only Pipeline

Groovy
pipeline { agent { kubernetes { yaml """ apiVersion: v1 kind: Pod spec: containers: - name: python image: ci/python-runner:1.0 command: ['cat'] tty: true """ } } stages { stage('Run Python Script') { steps { container('python') { sh 'python scripts/run.py' } } } } }

How Pipelines Work

1. Pod Creation

Jenkins creates a Kubernetes pod with specified containers

2. Stage Execution

Each stage runs in a specific container (or multiple containers)

3. Build Output

Output is captured and displayed in Jenkins console

4. Cleanup

Pod is automatically deleted when build completes

Important Configuration

command: ['cat'] and tty: true

These keep the container alive so Jenkins can execute commands inside it.

Without these: Container exits immediately and pipeline fails
Container Naming

Names in YAML must match container('name') calls in pipeline

container('python') { sh 'python script.py' }

Manage Docker Images

List, monitor, and safely clean up Docker images

List Images

View all Docker images
docker images

Filter only CI images:

docker images | grep ci/

Clean Up Process

Important: Never delete images in use by running containers
  1. Check running containers: docker ps
  2. Identify unused images
  3. Remove safe images: docker rmi IMAGE_NAME
  4. Verify cleanup: docker images

Safe Images to Keep

Image Pattern Reason
docker/desktop-* Docker Desktop system images
jenkins/jenkins:* Jenkins itself (if you use it)
registry.k8s.io/* Kubernetes system images
ci/python-runner Your custom CI images
Any running container image Currently in use

Remove Unused Image

# Normal removal docker rmi python:3.10-slim # Force removal (use with caution) docker rmi -f python:3.10-slim

Automated Cleanup

# Remove dangling images docker image prune # Aggressive cleanup (CAUTION) docker image prune -a
Warning: -a removes all unused images

Troubleshooting

Check if port 8080 is in use:

lsof -i :8080

Kill the process:

kill -9 PID

Then restart Jenkins

Verify Kubernetes status:

kubectl get nodes

If it fails, enable Kubernetes in:

Docker Desktop Settings → Kubernetes → Enable Kubernetes

Check Jenkins logs:

docker logs jenkins

Verify Kubernetes connectivity:

kubectl cluster-info

Verify image exists:

docker images | grep ci/

If missing, rebuild:

cd ci-images ./build-images.sh
